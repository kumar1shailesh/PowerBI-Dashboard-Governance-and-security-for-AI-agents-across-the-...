# PBIT caveats

`AI-Agent-Governance.pbit` is generated by
`scripts/build_pbit.py`, which assembles the PBIT internal structure
from the text artifacts in this repo and ZIPs it together.

## Why a `.pbit` instead of a `.pbix`

`.pbix` contains a compiled binary `DataModel` that only Power BI
Desktop's Analysis Services engine can write. `.pbit` (template)
stores the TMSL schema as JSON and Power BI Desktop builds the binary
model when you open the file. That JSON we *can* author from text.

## What's inside

- **`DataModelSchema`** — UTF-16 LE encoded TMSL. 10 tables + 13
  relationships + 74 measures + a `DataFolder` query parameter.
- **`Report/Layout`** — UTF-16 LE encoded report definition. One page
  ("Executive Summary") with 4 KPI cards + a header textbox.
- **`Connections`** — points the report at the embedded model.
- **`Metadata/Version`, `Version`, `Settings`, `SecurityBindings/Version`,
  `[Content_Types].xml`** — required manifest files.

## Known limitations

The PBIT format is poorly documented for from-scratch authoring. The
build script targets a reasonable common-denominator structure, but
Power BI Desktop versions vary in what they accept.

If `AI-Agent-Governance.pbit` fails to open:

1. **Inspect the error.** Power BI Desktop usually surfaces a specific
   message (e.g. "Unable to load data model" or "Schema version
   mismatch").
2. **Try the manual build.** [`pbix_build_guide.md`](../pbix_build_guide.md)
   takes ~30 minutes and produces a reliable result.
3. **Or use pbi-tools.** `pbi-tools` (https://pbi.tools/) is the
   open-source community CLI for round-tripping PBIX files. The CLI is
   primarily an extract → edit → compile flow on existing PBIX files.

## Regenerating

```powershell
python scripts/build_pbit.py
```

Rebuilds the PBIT from current text artifacts. Safe to re-run.

## Things the auto-generated PBIT does NOT have

- Pages 2–9 — only Executive Summary has visuals.
- Conditional formatting rules — those need to be added in Desktop.
- What-if parameters (`HourlyRateUSD`, `TeamCapacityHrsPerWeek`) — easier
  to add via Power BI Desktop's "New parameter" dialog.
- Row-level security — set up via `docs/rls_setup.md` after opening.
